# Review Log

## Purpose

This document records the review history of project deliverables throughout the project lifecycle.

Each completed task will be reviewed by the Project Lead before the project proceeds to the next task.

The purpose of this document is to:

- Ensure deliverables satisfy predefined acceptance criteria.
- Record review decisions and required revisions.
- Capture lessons learned during the project.
- Maintain an audit trail of project quality reviews.

This document follows an append-only approach. New reviews are always added to the end of the file. Existing review records should not be modified except to correct factual errors.

---

# Review Status Definition

| Status | Meaning |
|---------|---------|
| 🟢 Approved | Deliverable satisfies all acceptance criteria. The project may proceed to the next task. |
| 🟡 Approved with Minor Revisions | Deliverable is acceptable. Minor improvements are recommended but do not block subsequent work. |
| 🔴 Rejected | Deliverable does not satisfy the acceptance criteria. Rework is required before the project can continue. |

---

# Review Workflow

The review process follows the sequence below.

```
Task Assignment
        ↓
Task Implementation
        ↓
Deliverable Submission
        ↓
Project Lead Review
        ↓
Decision
        ↓
Approved / Approved with Minor Revisions / Rejected
        ↓
Lessons Learned
        ↓
Next Task
```

# Document Rules

1. Only the Project Lead updates review decisions.
2. The project may proceed only after the latest deliverable has been reviewed.
3. Every review must receive a unique Review ID (RV-001, RV-002, ...).
4. A task may have multiple review records if rework is required.
5. Review records are append-only and should not be deleted.
